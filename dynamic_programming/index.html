Common Library Functions

functools.lru_cache
Auto-memoizes function calls. Caches return values by input. None = unlimited size.
from functools import lru_cache @lru_cache(None) def my_func(n): ...
math.inf
Represents infinity. Used to initialize distances so any real path is automatically shorter.
import math dist = [math.inf] * n # any real number < math.inf
numpy.zeros / zeros_like
Initialize arrays of zeros. zeros_like(V) creates array with same shape as V.
import numpy as np V = np.zeros(n_states) newV = np.zeros_like(V)
numpy.sum
Sums all elements. Used to compute expected value: Σ P(s'|s,a)·V(s') in one line.
np.sum(P[s,a] * V) # elementwise multiply, then sum # = Σ P(s'|s,a) * V(s')
min() with multiple args
Returns smallest value. In edit distance, picks the cheapest operation.
min( dp[i-1][j], # deletion dp[i][j-1], # insertion dp[i-1][j-1] # substitution )
List comprehension (2D table)
Creates a 2D list (matrix). Outer loop = rows, inner = columns. Do NOT use [[0]*(n+1)]*(m+1) — it shares references!
# CORRECT — independent rows dp = [[0] * (n+1) for _ in range(m+1)] # WRONG — all rows are same object dp = [[0] * (n+1)] * (m+1)

Fibonacci — Three Versions

fib_memo(n) — Top-Down Memoization O(n) time O(n) space Top-Down lru_cache
DP Framework
State:dp[n] = F(n)
Recurrence:F(n) = F(n-1) + F(n-2)
Base cases:F(0)=0, F(1)=1
1from functools import lru_cacheImport the memoization decorator from standard library
2
3@lru_cache(None)▶ Caches every (n) → result. Next call with same n returns instantly
4def fib_memo(n):Function definition — identical body to naive version

6 return nF(0)=0, F(1)=1 — returns n directly since n≤1
7 return fib_memo(n-1) + fib_memo(n-2)▶ Recurrence — but cached! Each unique n computed only once
fib_bottom_up(n) — Tabulation O(n) time O(n) space Bottom-Up
1def fib_bottom_up(n):

3 return nHandle base cases before building table
4 dp = [0] * (n + 1)▶ Table of size n+1 — index 0 through n. All start at 0
5 dp[1] = 1Base case: F(1)=1. dp[0] already 0 from initialization

Staircase Problem

climb_stairs(n) — Steps of 1 or 2 O(n) time O(n) space Bottom-Up
DP Framework
State:dp[i] = ways to reach step i
Recurrence:dp[i] = dp[i-1] + dp[i-2]← came from i-1 OR i-2
Base cases:dp[0]=1, dp[1]=1← 1 way to stand at bottom
1def climb_stairs(n):
2 dp = [0] * (n + 1)Table with n+1 slots (index 0 to n)
3 dp[0] = dp[1] = 1▶ dp[0]=1: one way to "be" at bottom. dp[1]=1: one way to reach step 1
4 for i in range(2, n + 1):
5 dp[i] = dp[i-1] + dp[i-2]▶ Ways to reach i = ways from i-1 (1-step) + ways from i-2 (2-step)
6 return dp[n]
climb_stairs_3(n) — Steps of 1, 2, or 3 O(n) time O(n) space Bottom-Up
What Changed
Recurrence:dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
Base cases:dp[0]=1, dp[1]=1, dp[2]=2← extra base case needed!
Loop start:range(3, n+1)← starts at 3 not 2
1def climb_stairs_3(n):
2 if n <= 1: return 1
3 dp = [0] * (n + 1)
4 dp[0] = 1; dp[1] = 1; dp[2] = 2⚠ dp[2] needed! Without it, dp[2-3]=dp[-1] reads wrong element in Python
5 for i in range(3, n + 1):▶ Loop starts at 3 — all smaller cases already handled
6 dp[i] = dp[i-1] + dp[i-2] + dp[i-3]▶ Three sources now — can arrive from 3 different prior steps
7 return dp[n]

Bellman–Ford Algorithm

bellman_ford(edges, n, source) O(V·E) time O(V) space Iterative DP
DP Framework
State:dist[v] = shortest distance from source to v
Recurrence:dist[v] = min(dist[u] + w) for all edges u→v
Base cases:dist[source]=0, all others=∞
Iterations:n-1 times← longest path has at most n-1 edges
1import math
2
3def bellman_ford(edges, n, source):edges = list of (u, v, w) tuples. n = num nodes
4 dist = [math.inf] * n▶ Initialize all distances to ∞ — we know nothing yet
5 dist[source] = 0Base case: cost to reach yourself = 0
6 for _ in range(n - 1):⚠ Outer loop runs n-1 times — guarantees paths up to n-1 edges long
7 for u, v, w in edges:Inner loop: check EVERY edge each iteration. u=from, v=to, w=weight
8 if dist[u] + w < dist[v]:▶ "Can I reach v cheaper by going through u?" This is called RELAXATION
9 dist[v] = dist[u] + wYes — update to the shorter distance
10 return distReturns array: dist[i] = shortest distance from source to node i

Value Iteration (MDP)

value_iteration(P, R, gamma, iters) O(iters·S·A) time O(S) space Iterative DP numpy
Bellman Equation (the recurrence)
V(s) =max_a [ R(s,a) + γ · Σ P(s'|s,a) · V(s') ]
Inputs:P[s,a] = transition probs vector, R[s,a] = reward scalar
γ (gamma):0=shortsighted, 1=far-sighted, 0.9=typical
1import numpy as np
2
3def value_iteration(P, R, gamma=0.9, iters=50):P shape: [S,A,S]. R shape: [S,A]. gamma: discount factor
4 n_states, n_actions, _ = P.shapeUnpack dimensions. _ discards the third dimension (S again)
5 V = np.zeros(n_states)▶ Base case: assume all states worth 0 to start
6 for _ in range(iters):Repeat until values converge (like n-1 iterations in Bellman-Ford)
7 newV = np.zeros_like(V)⚠ Use newV not V — update all states simultaneously, not one-by-one
8 for s in range(n_states):Visit every state
9 action_values = []Collect value of each action at state s
10 for a in range(n_actions):Try every possible action
11 action_values.append(
12 R[s,a] + gamma * np.sum(P[s,a] * V)▶ THIS IS THE BELLMAN EQUATION: reward + discounted future value
13 )P[s,a]*V = elementwise multiply → np.sum = Σ P(s'|s,a)·V(s')
14 newV[s] = max(action_values)Take best action's value — the "max_a" part of Bellman equation
15 V = newVReplace old values — repeat until stable
16 return VV[s] = optimal value of each state

Edit Distance (Levenshtein)

edit_distance(s, t) O(m·n) time O(m·n) space 2D Bottom-Up
DP Framework
State:dp[i][j] = edit distance between s[:i] and t[:j]
Match:dp[i][j] = dp[i-1][j-1]← free, chars already equal
No match:dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
Base cases:dp[i][0]=i (deletions), dp[0][j]=j (insertions)
1def edit_distance(s: str, t: str) -> int:
2 m, n = len(s), len(t)m = len of s (rows), n = len of t (columns)
3 dp = [[0] * (n + 1) for _ in range(m + 1)]▶ 2D table (m+1)×(n+1). Extra row/col for empty string base cases
4 for i in range(m + 1):
5 dp[i][0] = i▶ First column: transform s[:i] to "" needs i deletions
6 for j in range(n + 1):
7 dp[0][j] = j▶ First row: transform "" to t[:j] needs j insertions
8 for i in range(1, m + 1):Outer loop: each character of s
9 for j in range(1, n + 1):Inner loop: each character of t
10 if s[i - 1] == t[j - 1]:⚠ s[i-1] not s[i] — dp[i] means i chars, so current char is index i-1
11 dp[i][j] = dp[i - 1][j - 1]▶ Match → FREE. Inherit diagonal — no operation needed
12 else:
13 dp[i][j] = 1 + min(No match → pick cheapest operation (+1 cost)
14 dp[i - 1][j],▶ DELETION — came from above (removed char from s)
15 dp[i][j - 1],▶ INSERTION — came from left (inserted char into s)
16 dp[i - 1][j - 1]▶ SUBSTITUTION — came from diagonal (swapped char)
17 )
18 return dp[m][n]Bottom-right cell = answer for full strings s and t

The Universal DP Pattern

Apply this framework to every DP problem Exam Strategy
Step 1 — State
What does dp[i] mean?
Step 2 — Recurrence
dp[i] = f(dp[i-1], ...)
Step 3 — Base Cases
Smallest known answers
Step 4 — Direction
Top-down or Bottom-up
Step 5 — Answer
dp[n] or dp[m][n]
Step 6 — Complexity
States × Work per state
