01

Single Bayesian Update Core update formula — prior × likelihood → normalise → posterior

numpy
np.array([v1, v2, v3])
Creates a NumPy array. Here used to store prior/likelihood values per hypothesis. Enables element-wise math.
np.sum(array)
Sums all elements. Used as the normaliser = P(D) — total probability of the data across all hypotheses.
a * b  (element-wise)
NumPy arrays multiply element-by-element. likelihood[i] × prior[i] for each hypothesis i simultaneously.
array.copy()
Creates an independent copy. Critical: prevents the posterior from aliasing the prior — modifying one would otherwise change the other.

# ── Observe H: update beliefs ────────────────────
10

element-wise: score each hypothesis
11

12
13
print(f"After H: {posterior}")
# [0.333, 0.667, 0.000]

Pattern breakdown

L10
likelihood * prior — NumPy element-wise multiply. Each hypothesis gets its raw score: "how probable is this evidence IF this hypothesis were true, weighted by my prior belief?"
L11
/ np.sum(posterior) — This is dividing by P(D), the total probability of the data. Forces all posteriors to sum to 1. Without this step you have unnormalised scores, not probabilities.
⚠️
Common mistake: Forgetting to normalise. likelihood * prior gives you the numerator only. Always divide by np.sum(likelihood * prior) to get valid probabilities.
02

Sequential Bayesian Updates Observe multiple data points one-by-one — posterior becomes new prior each step

numpy
list[likelihood_h, likelihood_t, ...]
A Python list of NumPy arrays. Each element is a likelihood vector for one observation. Loop through to update sequentially.
prior.copy()
Always copy before the loop. Without .copy(), posterior and prior point to the same object — modifications break both.
bayesian_sequential_update.py
1
# Observe: H, H, H, T — update after each flip
2
observations = [likelihood_h, likelihood_h, likelihood_h, likelihood_t]
list of likelihood vectors, 1 per flip
3
4
posterior = prior.copy()
copy — never alias prior directly
5
6
for obs in observations:
loop over each observation
7
posterior = obs * posterior
posterior IS the new prior each step
8
posterior = posterior / np.sum(posterior)
normalise after EVERY update
9
10
print(posterior)
# [1.0, 0.0, 0.0] after HHHT

Pattern breakdown

L2
observations list — each element is the full likelihood vector for that observation. The order matters: HHHT ≠ THHH in intermediate steps (same final result with enough data).
L4
prior.copy() — creates an independent array. If you write posterior = prior without .copy(), both names point to the same memory. Updating posterior silently corrupts prior.
L7
obs * posterior — note: posterior is now acting as the prior for this step. This is the Bayesian insight: yesterday's posterior is today's prior. The loop encodes this recycling.
L8
Normalise inside the loop — must normalise after every single update, not just at the end. Otherwise numerical errors accumulate and intermediate posteriors are invalid.

Monty Hall Simulation Frequentist simulation to verify the 2/3 switching result empirically

numpy
np.random.randint(1, 4)
Random integer in [1, 3] — note upper bound is exclusive. Used to randomly place car and player's pick.
np.random.choice(list)
Uniformly picks one element from a list. Used to randomly select which valid door the host opens when multiple are available.
[d for d in [...] if condition]
List comprehension filter. Used twice: once to find valid host doors, once to find the switch target door.
filtered_list[0]
Unpack a single-item list. After filtering to one remaining door, [0] extracts it from the list wrapper.
monty_hall_simulation.py

list comprehension: host's valid doors

host picks one randomly from valid

win only if original pick = car

17

[0] unpacks the 1-item list

Pattern breakdown

L11
Host constraint list comprehension — two conditions: d != pick (host can't open your door) AND d != car (host never reveals the car). This models the host's informed, constrained behaviour.
L19–20
Switch target — remove pick AND opened → exactly one door remains. [0] extracts it from the list. If you forget [0], you get a list, not an int, and the == comparison fails silently.
L15–22
Both strategies in one loop — elegant efficiency. The same trial evaluates stay AND switch simultaneously, so you don't need two separate simulation loops.
⚠️
Off-by-one: np.random.randint(1, 4) returns {1, 2, 3} — upper bound is EXCLUSIVE. Writing randint(1, 3) would only give {1, 2} and break the simulation.

Bayesian Regression — Posterior Cloud Frequentist single line vs. Bayesian distribution of plausible lines

numpy matplotlib.pyplot numpy.polynomial
np.linspace(start, stop, n)
Creates n evenly-spaced values between start and stop. Used to generate X axis for plotting smooth prediction lines.
np.random.normal(mean, std, n)
Samples n values from a Normal distribution. Used to simulate the Bayesian posterior over slope and intercept.
Polynomial.fit(X, y, deg)
Least-squares polynomial fit. Returns a Polynomial in Chebyshev basis — must call .convert() to get standard form.
.convert().coef
Converts to standard basis and returns [intercept, slope, ...]. Without .convert(), coefficients are in the wrong basis.
np.random.seed(42)
Sets the random seed for reproducibility. Same seed → same random numbers every run. Use before any random data generation.
plt.subplots(1, 2, figsize=(...))
Side-by-side comparison plots. fig = whole canvas, (ax1, ax2) = individual axes. Unpack directly when count is small.

Pattern breakdown

L12
Polynomial.fit().convert().coef — three chained calls. fit() works in Chebyshev space (numerically stable). convert() maps back to standard poly. coef = [intercept, slope] — intercept is index 0.
L16–17
np.random.normal(MLE, uncertainty, n) — the Bayesian "shortcut" in this notebook. Real Bayesian posterior uses MCMC (PyMC3). Here, Normal centred at MLE with a manually chosen std simulates the concept.
L30
alpha=0.1 — the cloud trick. 50 lines × 10% opacity. Where lines cluster densely, the colour intensifies. Cloud width = parameter uncertainty. Narrower cloud after more data = this width should shrink.
⚠️
Polynomial.fit() trap: .coef without .convert() gives Chebyshev coefficients, not standard [intercept, slope]. The numbers will be wrong and misleading.
