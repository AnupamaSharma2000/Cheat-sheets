Numerical Bayes — The Denominator Problem
posterior = (likelihood × prior) / evidence
Full pipeline: grid → likelihood → normalize → plot
core algorithm
📌 Compute the posterior numerically by multiplying likelihood × prior, then dividing by the area under the curve.
formula posterior(θ) = [ θˢ · (1−θ)ᶠ · prior(θ) ] / ∫ θˢ · (1−θ)ᶠ · prior(θ) dθ 
import numpy as np
import matplotlib.pyplot as plt

theta = np.linspace(0, 1, 500)          # 500 evenly spaced θ values

successes = 7
failures  = 3

likelihood = theta**successes * (1 - theta)**failures  # P(data|θ)

prior = np.ones_like(theta)                    # Uniform: P(θ) = 1 everywhere

unnormalized_posterior = likelihood * prior      # numerator of Bayes

evidence = np.trapezoid(unnormalized_posterior, theta)  # ∫ = normalizing constant

posterior = unnormalized_posterior / evidence   # divide → sums to 1

plt.plot(theta, prior, label="Prior")
plt.plot(theta, likelihood / likelihood.max(), label="Likelihood (scaled)")
plt.plot(theta, posterior, label="Posterior")
plt.legend()
plt.show()
→np.linspace(0,1,500)creates a grid of 500 θ values between 0 and 1. This is our "parameter space."
→theta**s * (1-theta)**fBernoulli likelihood — probability of seeing s successes and f failures given θ.
→np.ones_like(theta)uniform prior — every θ equally likely before seeing data.
→np.trapezoid(y, x)numerical integration (trapezoid rule) — computes the "area under the curve." This IS the evidence / normalizing constant.
→/ evidencedivides by the total area so the posterior sums to 1 (valid probability distribution).
→likelihood / likelihood.max()scales likelihood to [0,1] for visual comparison only — doesn't affect posterior math.
02
Beta-Bernoulli Conjugate Updating
closed-form posterior via parameter counting
Custom beta_pdf + sequential update loop
conjugate algorithm
📌 Beta prior + Bernoulli data → Beta posterior. Update rule: α += successes, β += failures.
update rule Beta(α, β) + data → Beta(α + Σxᵢ, β + Σ(1−xᵢ))
posterior mean = α / (α + β) 
theta = np.linspace(0.001, 0.999, 500)   # avoid log(0) at endpoints

def beta_pdf_numeric(theta, a, b):
    unnormalized = theta**(a-1) * (1-theta)**(b-1)    # Beta kernel
    Z = np.trapezoid(unnormalized, theta)           # normalizing constant
    return unnormalized / Z

a, b = 1, 1                              # start: Beta(1,1) = uniform prior
data = [1,1,1,0,1,0,1,1,1,1]             # 1=success, 0=failure

for x in data:
    a += x                               # success → increment α
    b += (1-x)                            # failure → increment β

# posterior is now Beta(9, 3)
# posterior mean = 9/(9+3) = 0.75
→0.001 to 0.999slightly inset from 0/1 to prevent θ⁰⁻¹ = θ⁻¹ (division by zero for a=1 or b=1 edge cases).
→theta**(a-1) * (1-theta)**(b-1)the "kernel" of the Beta distribution — the shape without the normalizing constant B(a,b).
→a, b = 1, 1Beta(1,1) = Uniform(0,1). Think of it as: 1 phantom success + 1 phantom failure seen before any data.
→for x in data: a += xeach 1 (success) adds to α. Each 0 (failure) adds to β via (1-x). This IS the closed-form Bayesian update.
→Result: Beta(9,3)posterior mean = 9/12 = 0.75. Prior was 1/2. Data pulled belief upward (8 successes, 2 failures).
03
MAP Estimation — Regularized Optimization
MAP = argmax(log-likelihood + log-prior)
Log-space MAP with argmax + axvline
MAP algorithm
📌 Add log-prior to log-likelihood → MAP is regularized MLE. Beta(2,2) prior shrinks estimate toward 0.5.
key identity MAP = argmax_θ [ log P(data|θ) + log P(θ) ]
Gaussian prior P(θ) → Ridge (L²) penalty
Laplace prior P(θ) → Lasso (L¹) penalty 
successes, failures = 7, 3

log_likelihood = successes*np.log(theta) + failures*np.log(1-theta)
# = log(θ⁷ · (1-θ)³) — Bernoulli log-likelihood

log_prior = (2-1)*np.log(theta) + (2-1)*np.log(1-theta)
# = log(Beta(2,2)) kernel — peaks at θ=0.5

log_posterior = log_likelihood + log_prior
# addition in log space = multiplication in original space

theta_mle = theta[np.argmax(log_likelihood)]   # peak of likelihood ≈ 0.70
theta_map = theta[np.argmax(log_posterior)]    # peak of posterior ≈ 0.67

plt.axvline(theta_mle, linestyle='--', label=f"MLE ≈ {theta_mle:.2f}")
plt.axvline(theta_map, linestyle='--', label=f"MAP ≈ {theta_map:.2f}")
→s*log(θ) + f*log(1-θ)log of Bernoulli likelihood. Log turns multiplication into addition — numerically stable and easier to optimize.
→(a-1)*log(θ) + (b-1)*log(1-θ)log of Beta kernel. With a=b=2, this adds log(θ) + log(1-θ) — a penalty that's 0 at θ=0.5 and -∞ at the extremes.
→log_posterior = log_L + log_priorthis is Bayes' theorem in log space. Adding logs = multiplying probabilities. No normalization needed for argmax.
→np.argmax(arr)returns the INDEX of the maximum value. Then theta[index] gives the θ value at that peak.
→plt.axvline(x)draws a vertical line at a specific x-value. Used to mark MLE and MAP on the plot.
04
Numerical Posterior — Non-Conjugate Prior
bayes still works, just compute it
Likelihood function + strange prior + full normalization
general pattern
📌 When prior isn't conjugate, define it manually, multiply by likelihood, normalize with trapezoid. Same 3-step recipe as always.
theta = np.linspace(0, 1, 2000)          # finer grid for accuracy

def likelihood(theta, data):
    s = sum(data)
    f = len(data) - s
    return theta**s * (1-theta)**f           # Bernoulli likelihood

data = [1,1,1,0,1,0,1,1,1,1]

prior = np.sqrt(theta*(1-theta))             # √(θ(1-θ)) — not a named family!
prior[np.isnan(prior)] = 0                   # fix NaN at θ=0 and θ=1

unnormalized = likelihood(theta, data) * prior  # P(data|θ) · P(θ)
posterior = unnormalized / np.trapezoid(unnormalized, theta)

plt.plot(theta, prior/prior.max(), label="Prior (scaled)")
plt.plot(theta, posterior, label="Posterior (numerical)")
→linspace(0,1,2000)2000 points instead of 500 — finer grid = more accurate numerical integration.
→def likelihood(theta, data)wraps the likelihood as a reusable function. Counts successes/failures from data automatically.
→np.sqrt(theta*(1-theta))a custom, non-standard prior shape. Not Beta, not Gaussian — just a function we chose. Algebra won't give a clean posterior.
→prior[np.isnan(prior)] = 0at θ=0 or θ=1, sqrt(0·1)=0 but floating point can produce NaN. This masks those safely to 0.
→same 3 linesunnormalized = L × P, then divide by trapezoid. This recipe works for ANY prior — conjugate or not.

The Universal Bayesian Recipe
algorithm
📌 Every posterior computation in this notebook follows the same 3-line pattern.
# Step 1: define your prior over the grid
prior = ...any function of theta...

# Step 2: multiply likelihood × prior (unnormalized posterior)
unnormalized = likelihood(theta, data) * prior

# Step 3: normalize by area → valid probability distribution
